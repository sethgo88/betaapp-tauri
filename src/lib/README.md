# src/lib

Shared singletons and utilities. Nothing in this folder contains JSX or feature-specific logic.

---

## db.ts ŌĆö SQLite client

```ts
import { getDb } from '@/lib/db'

const db = await getDb()
const rows = await db.select<MyType[]>('SELECT * FROM my_table WHERE id = ?', [id])
await db.execute('INSERT INTO my_table (id, name) VALUES (?, ?)', [id, name])
```

`getDb()` is lazy ŌĆö it opens the database connection once, runs `initSchema()` to ensure all tables exist, then returns the same instance on every subsequent call.

### initSchema()

Runs `CREATE TABLE IF NOT EXISTS` and `CREATE TRIGGER IF NOT EXISTS` for every table on each cold start. Safe to call multiple times (idempotent).

**Adding a new column** ŌĆö append a try/catch `ALTER TABLE` after the existing statements:

```ts
try {
    await db.execute(`ALTER TABLE climbs ADD COLUMN notes TEXT`)
} catch {
    // Column already exists ŌĆö safe to ignore
}
```

**Adding a new table** ŌĆö append a `CREATE TABLE IF NOT EXISTS` block to `initSchema()`.

### Rules
- Always use `?` positional parameters ŌĆö never string interpolation (SQL injection)
- Always filter `WHERE deleted_at IS NULL` on every read
- Never import `db.ts` directly in views or components ŌĆö only in `*.service.ts` files
- `updated_at` is maintained by DB trigger on UPDATE ŌĆö never set it on UPDATE from app code
- Dates: stored as `TEXT` in `datetime('now')` format (`YYYY-MM-DD HH:MM:SS`)

### Type conventions

| TypeScript | SQLite | Notes |
|---|---|---|
| `string` (UUID) | `TEXT` | All primary and foreign keys |
| `string` (datetime) | `TEXT` | `datetime('now')` format |
| `boolean` | `INTEGER` | 0 = false, 1 = true |
| `number` | `REAL` or `INTEGER` | |
| `string \| null` | `TEXT` | Nullable columns |

## supabase.ts ŌĆö Supabase client

```ts
import { supabase } from '@/lib/supabase'
```

Configured singleton using `@supabase/supabase-js` with the generated `Database` type from `database.types.ts`.

Credentials come from environment variables ŌĆö create a `.env` or `.env.local` in the project root (both are gitignored):

```
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
```

The client throws at import time if either variable is missing.

